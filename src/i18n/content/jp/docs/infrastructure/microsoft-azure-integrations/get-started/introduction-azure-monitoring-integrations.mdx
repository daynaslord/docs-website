当社の[Microsoft Azure](https://azure.microsoft.com/)インテグレーションは、Azureサービスをモニターし、そのサービスに関するデータをNew Relicにレポートすることで、アーキテクチャ全体の統合ビューを一元的に提供します。

Azureインテグレーションは、[Azure向けのAPMの.NETサポート](/docs/agents/net-agent/azure-installation)とは異なります。

## 要件 [#permissions]

個別のインテグレーションの要件については、[Azureインテグレーション文書](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list)をチェックしてください。

New Relicは、Azure Governmentにある、または[クラシックデプロイメントモデル](https://docs.microsoft.com/en-us/azure/azure-resource-manager/resource-manager-deployment-model)を通じ作成されたリソースから、監視データを取得できません。

## 機能

[Azureインテグレーションをアクティブ化](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations)すると、New Relicは標準の[ポーリング間隔](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-polling-intervals-infrastructure-integrations)にしたがって、Azureプラットフォームサービスのクエリを開始します。インテグレーションUIを使用して、次のことを行えます。

* エコシステムへの変更に応じて自動調整する[インテグレーションダッシュボード](/docs/infrastructure/integrations/find-use-infrastructure-integration-data)からパフォーマンスデータを表示する。
* Alertsで[アラート条件](/docs/infrastructure/new-relic-infrastructure/configuration/infrastructure-alerts-add-edit-or-view-host-alert-information#integration)を管理する。
* [データのクエリ](/docs/using-new-relic/data/understand-data/query-new-relic-data)。
